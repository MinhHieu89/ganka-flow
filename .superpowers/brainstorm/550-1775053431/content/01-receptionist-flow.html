Receptionist Module — Screen Flow

How the 4 screens and 3 popups connect together

Receptionist Workflow Map
/intake — Dashboard
KPI cards + Today's patient table + Search
Button: "Tiếp nhận BN mới"
/intake/new
Patient Intake Form
4 sections, full page
Save → back to Dashboard
Save & Pre-Exam → next module
Button: "Check-in" (on row)
Modal: Check-in
3 variants based on data
Complete → 1-click confirm
Incomplete → fill gaps first
Search → click result
Modal: Walk-in
Existing patient, new visit
Enter reason → create visit
Button: "Đặt lịch hẹn"
/schedule/new
Appointment Booking
Calendar + time slots, full page
New patient → minimal fields
Existing → read-only card
Patient Status Progression
Chưa đến → check-in → Chờ khám → doctor starts → Đang khám → done → Hoàn thành

Key Design Decisions

  • Dashboard modals — Check-in and walk-in are quick actions, stay as modals on /intake
  • Full pages — Intake form (/intake/new) and Appointment (/schedule/new) are detailed forms
  • Patient ID — GK-YYYY-NNNN format, auto-generated
  • Static mock data — No backend, all data is hardcoded for now
  • shadcn theme colors — Status badges adapted to neutral base CSS variables
